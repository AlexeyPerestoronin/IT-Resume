# Профессиональные интересы
По большому счёту, мне интересно всё что есть в IT (да и не только) в виду инженерных наклонностей характера и общей любви к технологиям.  
Однако, могу выделить некоторые сферы, которые интересуют меня особенно сильно:
* разработка классических настольных приложений (различные редакторы, САПР, CAD-системы, и т.д.);
* HFT - высокочастотный трейдинг;
* разработка B2B продуктов для настойки и управления инфраструктуры цикла создания ПО (CI/CD, Auto-Testing, системы контроля версий, т.д.).

# Личные проекты
Никогда не создавал в одиночку какие-то по настоящему значительные коммерческие решения, т.к. 99% процентов своих нужд и потребностей в ПО могу удовлетворить уже существующими open-source решениями.  
Тем не менее, на своём GitHub репозитории имею коллекцию ПО собственной разработки для закрытия ситуативных, специфических нужд:
* [System-Design](https://github.com/AlexeyPerestoronin/System-Design)  
    Собственная база знаний по System-Design, которую я переодически пополняю новыми интересными дизайнами ПО или другой тематической информацией.
* [AST](https://github.com/AlexeyPerestoronin/AST)  
    Универсальный шаблон для нового проекта, среди прочего полностью автоматизирующий процесс настройки рабочего окружения (скачал → запустил → готово)
* [Tasks-Cpp](https://github.com/AlexeyPerestoronin/Tasks-Cpp-)  
    Репозиторий на котором я переодически решаю алгоритмические задачи на C++
* [Tasks-Rust](https://github.com/AlexeyPerestoronin/Tasks-Rust-)  
    Репозиторий на котором я переодически решаю алгоритмические задачи на Rust (моём любимом ЯП)
* [MarkDownAutoTitle](https://github.com/AlexeyPerestoronin/MarkDownAutoTitle)  
    Поскольку я постоянно работаю с MarkDown документами, написал для себя утилиту для автоматического создания оглавления.
* [Cpp_TypesList](https://github.com/AlexeyPerestoronin/Cpp_TypesList)  
    В бытности работы в [Acronis](../JobPlaces/2020-2021_Acronis_Infoprotect.html) написал библиотеку для создания type-constructor-struct, которую было удобно использовать при программировании тейпов.
